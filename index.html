Vue源码学习日记·变化侦测篇（一）

Posted on 2021-08-06

这个系列日记主要记录一下自己学习Vue源码的过程，学习过程主要参考Vue中文社区的Vue源码解析文档，文档所分析源码版本有所不同，但代码逻辑差别不大，附上解析文档地址《Vue源码系列》，以及Vue源码地址链接：https://github.com/vuejs/vue。

变化侦测

Vue的最大特点之一就是数据驱动视图，视图是用户可以直观的看到的页面，现代的Web应用不仅仅是静态的，而是可以改变，这种改变可以是页面的外观，如页面的背景、颜色、文字大小等等，也可以是页面所显示的信息的更改。所谓数据驱动视图，就是由于用户的操作或后端的变化所导致的数据改变，在数据改变的同时，页面也随之而变化。

而页面如何根据数据的更改而变化呢？在数据驱动视图的模式中，数据与页面的关系就像一个函数：y = f(x)，x就是传入函数的数据及数据更改信息，y就是数据通过函数所得的页面结果。Vue就是其中的f()函数，Vue发现数据发生改变后，经过一系列的操作最后将结果反应在页面上。

首先，必须是数据发生了改变，页面才会进行更改，那么Vue是如何知道数据发生改变了呢？这就涉及到了Vue中的变化侦测，即是去追踪数据的状态，当数据变化时，通过数据变化的信息，立即去更改页面。Vue涉及到变化侦测部分的代码位于src>core>observer文件夹内。

依赖收集

通过使Object数据变得“可观测”，我们就可以随时知道数据在什么时候发生了变化，那么我们在知道一个数据发生变化之后，又怎么知道哪些页面依赖这个数据，会因为这个数据产生更改，以及什么时候去通知这些依赖该数据的页面进行更新呢？这就是Vue依赖收集部分所解决的问题。

什么时候收集依赖？什么时候通知依赖更新？

一个依赖在使用一个可观测Object数据的时候会获取该数据的值，即是会触发getter，所以当依赖使用某个数据，触发数据的getter的时候，说明该数据被这个依赖所依赖，就可以在getter中收集该依赖。然后该数据发生改变时，会触发某个setter，就可以在这个setter里面去通知依赖这个数据的所有依赖进行更新。

将依赖收集到哪里？

在能够收集依赖后，我们要把依赖信息收集到哪里来进行方便的管理以及在数据改变时通知依赖更新呢？

Vue通过为每一个数据建立一个依赖管理器，即Dep类的实例来对数据的所有依赖进行管理（源码文件位于src>core>observer>dep.js）。

现在我们让数据变的可观测了，并且知道了如何收集、管理依赖，并在数据变化的时候通知依赖更新，那么这个依赖究竟是什么？什么东西用到了数据，什么就是依赖，那什么东西需要用到数据，我们一般想的是页面或者说是视图，那这个依赖在Vue中是怎么用代码展示的呢？

Vue中实现了一个Watcher类（源码位于src>core>observer>watcher.js）。谁用到了数据，谁就是一个依赖，Vue就会为这个依赖创建一个Watcher实例，数据的Dep实例中收集、管理的就是每个依赖对应的Watcher实例，当数据发生变化时，Vue不直接通知依赖，而是通知依赖所对应的Watcher实例，再由Watcher实例去通知其所对应的真正的视图去跟新。下面是Watcher类的构造函数代码及get()方法代码：

谁用到了数据，谁就是一个依赖，Vue就会为这个依赖实例化一个Watcher对象。

当实例化Watcher类时，会先执行其构造函数，在构造函数中调用了this.get()实例方法，在get()方法中，首先通过pushTarget(this)把实例自身赋给了Dep类中的一个静态属性target上，然后通过value = this.getter.call(vm, vm)获取一下被依赖的数据，获取被依赖数据的目的是触发该数据上面的getter，然后在被触发的getter里会调用dep.depend()收集依赖，而在dep.depend()中取到挂载Dep.target上的值,通过Dep.target.addDep(this)调用Watcher实例上的方法检测该Watcher实例是否已经挂载到了这个Dep对象上，如果没有就调用dep.addSub(this)将其存入依赖数组中，在get()方法最后通过popTarget()将当前Dep.target移除（实际在src>core>observer>dep.js文件中声明了一个targetStack数组用于存储目标Watcher实例，并使Dep.target的值一直为数组的最后一个值，pushTarget(this)会将Watcher实例放入数组末尾，popTarget()会将数组长度减一，以达到添加移除当前目标依赖的目的）。

而当数据变化时，会触发数据的setter，在setter中调用了dep.notify()方法，在dep.notify()方法中，遍历所有依赖(即watcher实例)，执行依赖的update()方法，也就是Watcher类中的update()实例方法，在update()方法中调用数据变化的更新回调函数，从而更新视图。

虽然通过Object.defineProperty方法实现了对object数据的可观测，但是这个方法仅仅只能观测到object数据的取值及设置值，当我们向object数据里添加一对新的key/value或删除一对已有的key/value时，它是无法观测到的，导致当对object数据添加或删除值时，无法通知依赖，也就无法驱动视图进行响应式更新。

为了解决这一问题，Vue增加了两个全局API:Vue.set和Vue.delete。

总的来说，Vue变化侦测的大致思路就是：通过Object.defineProperty使得Object数据可观测化，Vue为此实现了一个Observer类，然后实现了一个依赖管理器Dep类，最后为每个需要使用数据的依赖实例化一个Watcher类，实例化的过程中，读取一次数据以触发getter向依赖管理器中添加自己，之后数据变化触发setter，通知Watcher对象，向相关依赖发送通知进行更新。

VsCode使用技巧

Posted on 2021-08-03

1. 安装步骤

在VsCode官网点击Download选择你电脑系统相应的版本点击下载，安装包下载完成后点击运行，勾选“确认”，点击下一步，选择要安装的位置，一直点击下一步然后点击安装即可。

2. 推荐插件

1. Chinese

该插件可以将VsCode的语言更改为中文，只需要在安装本插件后，根据弹出的提示重启VsCode即可。

2. Code Runner

原本的VsCode无法直接在软件上运行JS代码，在安装这个插件后，VsCode就获得了这个能力，可以直接运行JS脚本，而无需在浏览器上运行。

3. HTML Snippets

该插件可以自动提示HTML中的各类标签属性等，使得程序员无需在完整的输入长长的标签名或属性名。

4. open in browser

这个插件可以让你快速在默认的浏览器中打开你所选中的HTML文件。

5. Bracket Pair Colorizer

这个插件可以为你的代码中的括号加上不同的颜色，可以让你快速地看出相匹配的一对括号。

6. Path Intellisense

可以在你输入文件路径时提供智能提示。

3. 基础使用

1. 用户设置

打开文件-->首选项-->设置-->用户,即可进行用户个人设置，如软件颜色主题，字体大小等等。

2. 常用快捷键

  • 重开一行：光标在行尾的话，回车即可；不在行尾，ctrl + enter 向下重开一行；ctrl + shift + enter 则是在上一行重开一行

  • 删除一行：光标没有选择内容时，ctrl + x 剪切一行；ctrl + shift + k 直接删除一行

  • 移动一行：alt + ↑ 向上移动一行；alt + ↓ 向下移动一行

  • 复制一行：shift + alt + ↓ 向下复制一行；shift + alt + ↑ 向上复制一行

  • ctrl + z 回退

  • 选中一个词：ctrl + d

搜索或者替换：

  • ctrl + f ：搜索
  • ctrl + alt + f： 替换
  • ctrl + shift + f：在项目内搜索

  • 通过Ctrl + `可以打开或关闭终端

  • Ctrl + P 快速打开最近打开的文件

  • Ctrl + Shift + N 打开新的编辑器窗口

  • Ctrl + Shift + W关闭编辑器

  • Home 光标跳转到行头

  • End 光标跳转到行尾

  • Ctrl + Home 跳转到页头

  • Ctrl + End 跳转到页尾

  • Ctrl + Shift + [ 折叠区域代码

  • Ctrl + Shift + ]展开区域代码

  • Ctrl + / 添加关闭行注释

  • Shift + Alt +A 块区域注释

3. 打开项目文件

点击文件，在展开的下拉列表中点击打开文件夹，在打开的资源管理器中选择你需要编辑的项目的文件夹，点击打开。VsCode的左侧导航栏中即会显示当前打开的文件夹，包括其中的文件目录等。点击其中的文件即可在右侧的内容区编辑代码。

4. Git

如果打开的项目配置了Git，可以在图中所示的位置对更改的文件进行暂存和提交。

Chrome调试基础方法总结

Posted on 2021-08-03

1. 调试工具

在浏览器中通过按Ctrl+Shift+I或者F12可以打开开发者工具，我们调试过程中所需要的各种工具面板及调试信息都在这里。

1. Element

上图是开发者工具中的功能菜单，每个菜单都有相应的功能和使用方法，从左往右的第二个图标是设备图标，点击它可以切换到不同的终端进行开发模式，移动端和PC端的一个切换，可以选择不同的移动终端设备，同时可以选择不同的尺寸比例。

点击Elements菜单，可以查看，修改页面上的元素，包括DOM标签，以及CSS样式的查看，修改，还有相关盒模型的图形信息。下图我们可以看到当我鼠标选择id为id1的div元素时，下方的Styles栏会展示出此id的样式信息，此时可以进行修改，修改会立即在页面上生效。

2. Console控制台

Console控制台可以用于打印和输出相关的命令信息，除此之外，在调试器在断点暂停时，可以用console.log()来查看一些页面数据。

3. Sources

上图中的Sources菜单显示了js资源页面，在这个页面内我们可以找到当前浏览器页面中的js源文件，方便我们查看和调试。

4. Network

网络请求菜单可以看到所有的资源请求，包括网络请求，图片资源，html，css，js文件等请求，可以根据需求筛选请求项，一般多用于网络请求的查看和分析，分析后端接口是否正确传输，获取的数据是否准确。

点击请求名可查看相应的请求头信息及响应信息等。

2. 断点调试

1. 断点

在代码的某一处打上断点后，启动调试时，调试器会在断点处自动暂停JavaScript代码执行。

想要打上断点只需要在那一行的代码的行序上点击鼠标左键即可,点击右键可展示更多选项,比如编辑、删除、禁用断点等。

2. Watch

右侧的信息下拉列表中的Watch栏可以通过点击+号输入表达式。调试器会随时显示并在执行过程中更新它的值。

3. Call Stack

当执行到某一程序块的断点处停下来后，右侧调试区的Call Stack 会显示当前断点所处的方法调用栈，从上到下由最新调用处依次往下排列。

4. Scope

Scope栏可以显示当前的变量。

Git学习总结

Posted on 2021-08-03

1. Git

  1. 版本控制系统
  2. 将数据看作是对小型文件系统的一组快照

2. 基本操作

git status

该命令用于检查当前文件状态，如果当前目录具有未跟踪的新文件或已修改未暂存、已暂存未提交至本地仓库的文件，Git都会在此列出。除此之外，可以通过在git status命令后添加--s或short来获取简略格式的状态信息。

git add …

该命令用于跟踪新建文件或者用于暂存修改的文件。例如修改了README.md文件之后，运行git status命令Git会显示如下状态信息：

在状态信息中可以看到README.md被修改了，但并未暂存，并提示使用git add <file>...命令进行暂存。在运行git add README.md之后再次运行git status命令：

状态信息显示修改后的README.md文件已被暂存。

git commit

git commit命令用于将暂存区域的文件提交至本地仓库。单使用git commit会打开文本编辑器以输入本次提交的说明，可以在命令后面添加-m，并在同一行增加提交说明。除此以外，可以在git commit后面加上-a的选项来跳过使用git add的步骤。

3. 远程仓库

git clone [url]

该命令可以克隆远程仓库获得一份已经存在的Git仓库的拷贝。

git remote add

该命令用于在本地添加一个远程仓库。

git remote

该命令可以查看已经配置的远程仓库，并且可以加上-v选项，以显示远程仓库使用的Git保存的简写与其对应的url。其效果如下所示

git fetch [remote-name]

该命令可以访问远程仓库，从中拉取所有你还没有的数据。 执行完成后，你将会拥有那个远程仓库中所有分支的引用，可以随时合并或查看。

git push [remote-name] [branch-name]

该命令会将你所做的代码工作备份到远程仓库。

4. 分支

git checkout -b [branch-name]

这条命令可以新建一条分支并切换到该条新建分支上，相当于按顺序使用git branch [branch-name]和git checkout [branch-name]两条命令。

5. 查看提交历史

git log

可以使用git log命令来查看提交历史。可以在命令后添加其他的选项来设置历史信息的显示格式。

如-p,可以显示每次提交的内容差异，--stat可以只显示简略的提交信息，使用--graph可以添加一些ASCII字符串来形象地展示分支、合并历史，等等。

6. tag

git tag

可以使用这条命令以字母顺序列出已有的标签。

git show [tag-name]

使用这条命令可以看到标签信息以及对应的提交信息。

git tag [tag-name]

使用这条命令可以为最近的一次提交打上轻量标签。

git tag -a [tag-name] -m

这条命令可以为最近的一次提交打上附注标签，-m选项后面可以加上一条标签信息。

git tag -a [tag-name]

如果想要对过去的某次提交打标签可以使用这条命令，使用时需要在命令的后面加上想要打标签的提交的校验和。

git push [remote-name] [tag-name]

这条命令可以将标签显式地标签到远程仓库上。

7. 撤销操作

git commit –amend

如果在提交文件之后发现有文件忘记添加到暂存区，或是提交信息写错了，可以在添加遗忘的文件后使用git commit amend命令进行第二次提交，这次提交会代替上一次的提交。

git reset HEAD …

该命令可以撤销已经暂存的文件，使用时在命令后加上要撤销暂存的文件名。

git checkout – …

该命令可以撤销对文件的修改。
