VsCode使用技巧

1. 安装步骤

在VsCode官网点击Download选择你电脑系统相应的版本点击下载，安装包下载完成后点击运行，勾选“确认”，点击下一步，选择要安装的位置，一直点击下一步然后点击安装即可。

2. 推荐插件

1. Chinese

该插件可以将VsCode的语言更改为中文，只需要在安装本插件后，根据弹出的提示重启VsCode即可。

2. Code Runner

原本的VsCode无法直接在软件上运行JS代码，在安装这个插件后，VsCode就获得了这个能力，可以直接运行JS脚本，而无需在浏览器上运行。

3. HTML Snippets

该插件可以自动提示HTML中的各类标签属性等，使得程序员无需在完整的输入长长的标签名或属性名。

4. open in browser

这个插件可以让你快速在默认的浏览器中打开你所选中的HTML文件。

5. Bracket Pair Colorizer

这个插件可以为你的代码中的括号加上不同的颜色，可以让你快速地看出相匹配的一对括号。

6. Path Intellisense

可以在你输入文件路径时提供智能提示。

3. 基础使用

1. 用户设置

打开文件-->首选项-->设置-->用户,即可进行用户个人设置，如软件颜色主题，字体大小等等。

2. 常用快捷键

  • 重开一行：光标在行尾的话，回车即可；不在行尾，ctrl + enter 向下重开一行；ctrl + shift + enter 则是在上一行重开一行

  • 删除一行：光标没有选择内容时，ctrl + x 剪切一行；ctrl + shift + k 直接删除一行

  • 移动一行：alt + ↑ 向上移动一行；alt + ↓ 向下移动一行

  • 复制一行：shift + alt + ↓ 向下复制一行；shift + alt + ↑ 向上复制一行

  • ctrl + z 回退

  • 选中一个词：ctrl + d

搜索或者替换：

  • ctrl + f ：搜索
  • ctrl + alt + f： 替换
  • ctrl + shift + f：在项目内搜索

  • 通过Ctrl + `可以打开或关闭终端

  • Ctrl + P 快速打开最近打开的文件

  • Ctrl + Shift + N 打开新的编辑器窗口

  • Ctrl + Shift + W关闭编辑器

  • Home 光标跳转到行头

  • End 光标跳转到行尾

  • Ctrl + Home 跳转到页头

  • Ctrl + End 跳转到页尾

  • Ctrl + Shift + [ 折叠区域代码

  • Ctrl + Shift + ]展开区域代码

  • Ctrl + / 添加关闭行注释

  • Shift + Alt +A 块区域注释

3. 打开项目文件

点击文件，在展开的下拉列表中点击打开文件夹，在打开的资源管理器中选择你需要编辑的项目的文件夹，点击打开。VsCode的左侧导航栏中即会显示当前打开的文件夹，包括其中的文件目录等。点击其中的文件即可在右侧的内容区编辑代码。

4. Git

如果打开的项目配置了Git，可以在图中所示的位置对更改的文件进行暂存和提交。

Chrome调试基础方法总结

1. 调试工具

在浏览器中通过按Ctrl+Shift+I或者F12可以打开开发者工具，我们调试过程中所需要的各种工具面板及调试信息都在这里。

1. Element

上图是开发者工具中的功能菜单，每个菜单都有相应的功能和使用方法，从左往右的第二个图标是设备图标，点击它可以切换到不同的终端进行开发模式，移动端和PC端的一个切换，可以选择不同的移动终端设备，同时可以选择不同的尺寸比例。

点击Elements菜单，可以查看，修改页面上的元素，包括DOM标签，以及CSS样式的查看，修改，还有相关盒模型的图形信息。下图我们可以看到当我鼠标选择id为id1的div元素时，下方的Styles栏会展示出此id的样式信息，此时可以进行修改，修改会立即在页面上生效。

2. Console控制台

Console控制台可以用于打印和输出相关的命令信息，除此之外，在调试器在断点暂停时，可以用console.log()来查看一些页面数据。

3. Sources

上图中的Sources菜单显示了js资源页面，在这个页面内我们可以找到当前浏览器页面中的js源文件，方便我们查看和调试。

4. Network

网络请求菜单可以看到所有的资源请求，包括网络请求，图片资源，html，css，js文件等请求，可以根据需求筛选请求项，一般多用于网络请求的查看和分析，分析后端接口是否正确传输，获取的数据是否准确。

点击请求名可查看相应的请求头信息及响应信息等。

2. 断点调试

1. 断点

在代码的某一处打上断点后，启动调试时，调试器会在断点处自动暂停JavaScript代码执行。

想要打上断点只需要在那一行的代码的行序上点击鼠标左键即可,点击右键可展示更多选项,比如编辑、删除、禁用断点等。

2. Watch

右侧的信息下拉列表中的Watch栏可以通过点击+号输入表达式。调试器会随时显示并在执行过程中更新它的值。

3. Call Stack

当执行到某一程序块的断点处停下来后，右侧调试区的Call Stack 会显示当前断点所处的方法调用栈，从上到下由最新调用处依次往下排列。

4. Scope

Scope栏可以显示当前的变量。

Git学习总结

1. Git

  1. 版本控制系统
  2. 将数据看作是对小型文件系统的一组快照

2. 基本操作

git status

该命令用于检查当前文件状态，如果当前目录具有未跟踪的新文件或已修改未暂存、已暂存未提交至本地仓库的文件，Git都会在此列出。除此之外，可以通过在git status命令后添加--s或short来获取简略格式的状态信息。

git add …

该命令用于跟踪新建文件或者用于暂存修改的文件。例如修改了README.md文件之后，运行git status命令Git会显示如下状态信息：

在状态信息中可以看到README.md被修改了，但并未暂存，并提示使用git add <file>...命令进行暂存。在运行git add README.md之后再次运行git status命令：

状态信息显示修改后的README.md文件已被暂存。

git commit

git commit命令用于将暂存区域的文件提交至本地仓库。单使用git commit会打开文本编辑器以输入本次提交的说明，可以在命令后面添加-m，并在同一行增加提交说明。除此以外，可以在git commit后面加上-a的选项来跳过使用git add的步骤。

3. 远程仓库

git clone [url]

该命令可以克隆远程仓库获得一份已经存在的Git仓库的拷贝。

git remote add

该命令用于在本地添加一个远程仓库。

git remote

该命令可以查看已经配置的远程仓库，并且可以加上-v选项，以显示远程仓库使用的Git保存的简写与其对应的url。其效果如下所示

git fetch [remote-name]

该命令可以访问远程仓库，从中拉取所有你还没有的数据。 执行完成后，你将会拥有那个远程仓库中所有分支的引用，可以随时合并或查看。

git push [remote-name] [branch-name]

该命令会将你所做的代码工作备份到远程仓库。

4. 分支

git checkout -b [branch-name]

这条命令可以新建一条分支并切换到该条新建分支上，相当于按顺序使用git branch [branch-name]和git checkout [branch-name]两条命令。

5. 查看提交历史

git log

可以使用git log命令来查看提交历史。可以在命令后添加其他的选项来设置历史信息的显示格式。

如-p,可以显示每次提交的内容差异，--stat可以只显示简略的提交信息，使用--graph可以添加一些ASCII字符串来形象地展示分支、合并历史，等等。

6. tag

git tag

可以使用这条命令以字母顺序列出已有的标签。

git show [tag-name]

使用这条命令可以看到标签信息以及对应的提交信息。

git tag [tag-name]

使用这条命令可以为最近的一次提交打上轻量标签。

git tag -a [tag-name] -m

这条命令可以为最近的一次提交打上附注标签，-m选项后面可以加上一条标签信息。

git tag -a [tag-name]

如果想要对过去的某次提交打标签可以使用这条命令，使用时需要在命令的后面加上想要打标签的提交的校验和。

git push [remote-name] [tag-name]

这条命令可以将标签显式地标签到远程仓库上。

7. 撤销操作

git commit –amend

如果在提交文件之后发现有文件忘记添加到暂存区，或是提交信息写错了，可以在添加遗忘的文件后使用git commit amend命令进行第二次提交，这次提交会代替上一次的提交。

git reset HEAD …

该命令可以撤销已经暂存的文件，使用时在命令后加上要撤销暂存的文件名。

git checkout – …

该命令可以撤销对文件的修改。
