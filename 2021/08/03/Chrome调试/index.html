Chrome调试基础方法总结

发表于 2021-08-03

1. 调试工具

在浏览器中通过按Ctrl+Shift+I或者F12可以打开开发者工具，我们调试过程中所需要的各种工具面板及调试信息都在这里。

1. Element

上图是开发者工具中的功能菜单，每个菜单都有相应的功能和使用方法，从左往右的第二个图标是设备图标，点击它可以切换到不同的终端进行开发模式，移动端和PC端的一个切换，可以选择不同的移动终端设备，同时可以选择不同的尺寸比例。

点击Elements菜单，可以查看，修改页面上的元素，包括DOM标签，以及CSS样式的查看，修改，还有相关盒模型的图形信息。下图我们可以看到当我鼠标选择id为id1的div元素时，下方的Styles栏会展示出此id的样式信息，此时可以进行修改，修改会立即在页面上生效。

2. Console控制台

Console控制台可以用于打印和输出相关的命令信息，除此之外，在调试器在断点暂停时，可以用console.log()来查看一些页面数据。

3. Sources

上图中的Sources菜单显示了js资源页面，在这个页面内我们可以找到当前浏览器页面中的js源文件，方便我们查看和调试。

4. Network

网络请求菜单可以看到所有的资源请求，包括网络请求，图片资源，html，css，js文件等请求，可以根据需求筛选请求项，一般多用于网络请求的查看和分析，分析后端接口是否正确传输，获取的数据是否准确。

点击请求名可查看相应的请求头信息及响应信息等。

2. 断点调试

1. 断点

在代码的某一处打上断点后，启动调试时，调试器会在断点处自动暂停JavaScript代码执行。

想要打上断点只需要在那一行的代码的行序上点击鼠标左键即可,点击右键可展示更多选项,比如编辑、删除、禁用断点等。

2. Watch

右侧的信息下拉列表中的Watch栏可以通过点击+号输入表达式。调试器会随时显示并在执行过程中更新它的值。

3. Call Stack

当执行到某一程序块的断点处停下来后，右侧调试区的Call Stack 会显示当前断点所处的方法调用栈，从上到下由最新调用处依次往下排列。

4. Scope

Scope栏可以显示当前的变量。
